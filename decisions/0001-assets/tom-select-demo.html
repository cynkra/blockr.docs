Tom Select — styled to blockr design system

Four live Tom Select instances with the same CSS variables the blockr components use. Each demonstrates a feature we don't have in Blockr.Select today.

1. Single + secondary label (ADaM-style)

What dm_pull_block does today — identical UX, with richer label support.

2. Multi-select + drag-to-reorder

Matches Blockr.Select.multi with reorderable tags.

3. Grouped options (by source table)

Can't do this in Blockr.Select today. One of the triggers in ADR-0001.

4. Creatable tags

User types a value → it becomes a tag. Common filter/crossfilter request.

5. Virtualized: 10 000 options, smooth search + scroll

The ticker-catalog use case. Hand-rolled virtualization would be non-trivial; this is a plugin in Tom Select.

6. Rich two-line options with category badges (matches blockr.seasonal)

Same structure as Blockr.SelectRich: bold name + category pill on the right + description line below. Tom Select's render.option callback returns arbitrary HTML.
